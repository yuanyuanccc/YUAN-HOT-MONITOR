# YUAN HOT MONITOR 方案设计

## 目标

构建一个“热点发现 + AI 识别 + 人工确认”的系统，先完成网页版闭环，再扩展 Agent Skills。

## 产品闭环

1. 用户在网页端输入关键词与时间窗口
2. 系统抓取公开来源文本并做基础聚合
3. 调用 OpenRouter 大模型输出热点摘要、热度等级与风险提示
4. 用户人工确认热点状态并写入本地记录
5. 网页端展示结果历史，支持复盘

## 技术方案

- 前端：React + Vite + TypeScript
- 后端：Node.js + Express + TypeScript
- AI 接入：OpenRouter Chat Completions 接口
- 数据持久化：本地 JSON 文件（低门槛、便于快速落地）

## 核心页面

1. 指挥台首页：品牌化大标题、渐变视觉、动态热力背景
2. 热点识别面板：关键词、时间范围、样本输入、AI 识别按钮
3. 人工确认面板：确认通过、驳回、备注
4. 历史结果面板：按时间倒序展示处理记录

## 接口设计

- `POST /api/hotspots/analyze`
  - 入参：`keyword`、`timeRange`、`samples[]`
  - 出参：`title`、`summary`、`score`、`riskLevel`、`actions[]`

- `POST /api/hotspots/confirm`
  - 入参：`analysisId`、`status`、`note`
  - 出参：确认后的完整记录

- `GET /api/hotspots/history`
  - 出参：历史记录数组

## 交付节奏

1. 完成网页版前后端与 OpenRouter 接入
2. 完成功能联调与基础测试
3. 追加 Agent Skills（基于网页端稳定逻辑）

## 人工确认点

1. AI 分析结果是否符合业务预期
2. 热点状态确认流程是否满足审阅习惯
3. 页面视觉风格是否达到“独特”要求
